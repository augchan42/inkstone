# Audit Dimensions Reference

Evaluate screens in this order (highest impact first).

## 1. Visual Hierarchy

| Check | Pass Criteria |
|-------|---------------|
| Eye landing | Most important element is most prominent |
| 2-second test | User understands screen purpose immediately |
| Weight = importance | Visual prominence matches functional priority |
| Single focus | One clear primary action per screen |

## 2. Spacing & Rhythm

| Check | Pass Criteria |
|-------|---------------|
| Consistency | Same spacing tokens used throughout |
| Breathing room | Elements not cramped |
| Vertical rhythm | Harmonious flow down the page |
| Grouping | Related items closer, unrelated farther |

## 3. Typography

| Check | Pass Criteria |
|-------|---------------|
| Hierarchy | Clear size/weight differentiation |
| Restraint | ≤3 font sizes per screen |
| Readability | Comfortable line height (1.4-1.6×) |
| Calm vs chaos | Type feels organized, not competing |

## 4. Color

| Check | Pass Criteria |
|-------|---------------|
| Purpose | Every color has a reason |
| Restraint | Limited palette, not rainbow |
| Contrast | WCAG AA (4.5:1 body, 3:1 large text) |
| Guidance | Color directs attention appropriately |

## 5. Alignment & Grid

| Check | Pass Criteria |
|-------|---------------|
| Grid adherence | All elements on consistent grid |
| Pixel precision | Nothing off by 1-2px |
| Edge alignment | Left edges align, right edges align |

## 6. Components

| Check | Pass Criteria |
|-------|---------------|
| Consistency | Same component = same style everywhere |
| Affordance | Interactive elements look interactive |
| States | All states designed (default, pressed, disabled, focus) |

## 7. Iconography

| Check | Pass Criteria |
|-------|---------------|
| Cohesion | Single icon family/style |
| Size consistency | Same optical size throughout |
| Meaning | Icons support comprehension, not decoration |

## 8. Motion

| Check | Pass Criteria |
|-------|---------------|
| Purpose | Every animation has a reason |
| Physics | Feels natural, not jarring |
| Responsiveness | Immediate feedback on interaction |
| Reduce Motion | Respects accessibility setting |

## 9. States

| State | Check |
|-------|-------|
| Empty | Feels intentional, guides to first action |
| Loading | Consistent style, feels alive not frozen |
| Error | Helpful message, not hostile, styled consistently |
| Success | Clear confirmation, appropriate celebration level |

## 10. Dark Mode

| Check | Pass Criteria |
|-------|---------------|
| Designed | Not just inverted colors |
| Contrast | All text passes contrast in dark |
| Shadows | Adjusted or replaced with borders |
| Semantic colors | Still meaningful in dark context |

## 11. Density

| Check | Pass Criteria |
|-------|---------------|
| Necessity | Every element earns its place |
| Redundancy | Nothing says same thing twice |
| Removal test | Tried removing elements? |

## 12. Responsiveness

| Check | Pass Criteria |
|-------|---------------|
| Touch targets | Minimum 44×44pt |
| Fluid adaptation | Not just 3 breakpoint snaps |
| Thumb reach | Primary actions in easy thumb zone |

## 13. Accessibility

| Check | Pass Criteria |
|-------|---------------|
| VoiceOver | Logical reading order, meaningful labels |
| Dynamic Type | Scales appropriately |
| Focus states | Visible keyboard focus |
| Color independence | Info not conveyed by color alone |
